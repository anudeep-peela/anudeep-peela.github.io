## 2024-05-18 - Fitts's Law and the "Invisible" Tap Target
**Learning:** In responsive, card-based designs with interactive accordions or expanding sections, relying purely on a tiny 44x44px button icon (like `.case-expand-btn`) creates poor pointing accuracy issues for both mouse and mobile touch users, frustrating the experience. Adding explicit `:focus-visible` styling (rather than relying on native browser defaults) is vital to ensure keyboard navigation remains accessible.
**Action:** Always wrap the logic to the entire containing header (`.case-header`) via JavaScript click listeners, adding `cursor: pointer` to the header. Stop event propagation on the button itself to prevent double-toggling. Manually specify `:focus-visible` outline loops for interactive pseudo-buttons.
